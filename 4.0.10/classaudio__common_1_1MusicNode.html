Detailed Description

ROS 2 node that streams WAV music files by publishing audio chunks and exposes ROS 2 services to control playback.

Playback runs in a dedicated background thread (publish_thread_). Audio data is read from a WAV file using WaveFile and published as AudioStamped messages at a rate determined by the file's sample rate and the chunk size. The node supports optional looping as well as pause/resume without interrupting the publishing thread.

ROS 2 Parameters
  • chunk (int, default 2048): Number of float samples per published message.
  • frame_id (string, default ""): TF frame ID attached to published message headers.
